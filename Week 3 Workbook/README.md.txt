# Summary of What We Did and What We Learned

### Day 1 – Key Database Concepts  
We began Week 3 by exploring the core principles behind how databases work. This included understanding **primary keys**, **secondary keys**, and **foreign keys**, and how they are used to uniquely identify records and link tables together. We also looked at real‑world examples of **one‑to‑one**, **one‑to‑many**, and **many‑to‑many** relationships, helping us understand how data is structured in systems such as retail, education, and customer management.  
We then compared **relational** and **non‑relational** databases, learning when each model is appropriate. This helped us understand why SQL databases are used for structured, organised data, while NoSQL databases are better suited for flexible or unstructured information.

---

### Day 3 – Understanding SQL JOIN Types  
We researched the main SQL JOIN types and learned how they are used to combine data from multiple tables. This included:  
- **Inner Join**  
- **Left Join**  
- **Right Join**  
- **Full Join**  
- **Cross Join**  
- **Self Join**  

For each JOIN type, we explored what it does, how it behaves, and when it would be used in real datasets. This strengthened our understanding of how relational databases connect information and how SQL queries retrieve meaningful insights from linked tables.

---

### Day 4 – Designing a Database for a Retail Business  
We completed a written task where we designed a database for a small retail shop. This involved:  
- Identifying the business requirements  
- Determining what data needed to be stored (inventory, customers, sales, loyalty programme)  
- Designing a database schema with appropriate tables and relationships  
- Writing example SQL commands to create tables and define primary/foreign keys  
- Planning how to insert initial data using SQL INSERT statements  
- Considering maintenance, backups, and data security  

This task helped us understand the full lifecycle of database creation—from planning and design to implementation and long‑term management.

---

### Day 4 – SQL Practical Tasks  
We worked with the **world_db** database to complete a series of SQL queries based on real‑world scenarios. These included tasks such as:  
- Counting cities in the USA  
- Finding the country with the highest life expectancy  
- Listing cities containing the word “New”  
- Displaying the first 10 most populous cities  
- Identifying cities with populations above 2 million  
- Filtering cities by prefixes or population ranges  
- Sorting cities alphabetically  
- Finding the most and least populated cities  
- Identifying countries with the largest populations  
- Listing cities in Europe  
- Calculating average population by country  
- Comparing capital city populations  
- Identifying countries with low population density  
- Finding cities with high GDP per capita  
- Displaying rows 31–40 for deeper analysis  

These exercises strengthened our practical SQL skills and helped us understand how data analysts use SQL to answer business questions quickly and accurately.

---

## Overall Learning  
Throughout Week 3, we developed strong foundational skills in databases and SQL:

- Understanding how relational databases are structured  
- Knowing the purpose of primary, secondary, and foreign keys  
- Recognising different types of table relationships  
- Comparing relational and non‑relational database models  
- Learning how SQL JOINs combine data across tables  
- Designing a database schema for a real business scenario  
- Writing SQL commands to create, populate, and maintain tables  
- Running practical SQL queries to extract insights from large datasets  

These activities built confidence in working with structured data and prepared us for more advanced SQL analysis and database design in future weeks.
